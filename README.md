# W-LSTMix for Anomaly Detection

**Point-wise anomaly detection in building energy time series, adapted from the [W-LSTMix](https://github.com/shivDwd/W-LSTMix) forecasting model.**

This repository is an adaptation of the original W-LSTMix architecture вҖ” a lightweight hybrid model combining Wavelet decomposition, LSTM, and MLP-Mixer вҖ” repurposed from **time series forecasting** to **anomaly detection** on building energy consumption data.

---

## What Changed

The original W-LSTMix was designed for **short-term load forecasting**: given a historical window, it predicted future energy values. We adapted it for **anomaly detection** with the following key changes:

### 1. Labeling Strategy: Percentile-Based Anomaly Labels

Instead of predicting future values, each time step in the series receives a **binary label** (normal vs. anomalous). Labels are generated by:

1. **Decomposing** the time series into **trend** and **seasonality** components (via wavelet transform or seasonal decompose).
2. Computing **percentile-based boundaries** (upper and lower) on each component independently.
3. **Flagging** each point as **anomalous** (`1`) if it falls outside the percentile bounds in **either** component; otherwise it is labeled as normal (`0`).

This approach captures anomalies that manifest in different structural aspects of the signal вҖ” a sudden spike in the trend, or an unusual seasonal pattern вҖ” without requiring hand-crafted thresholds tied to raw values.

### 2. Architecture Changes

| Aspect | Original (Forecasting) | Adapted (Anomaly Detection) |
|--------|------------------------|-----------------------------|
| **Output** | Future values `[B, forecast_length]` | Per-point logits `[B, backcast_length]` |
| **Loss** | MSE / Huber Loss | BCEWithLogitsLoss |
| **Head** | Separate trend & season forecast heads | Shared classifier MLP over concatenated residuals |
| **Forecast branch** | Each block produces `(backcast, forecast)` | Each block produces only `backcast` (residual subtraction) |
| **Final output** | `trend_forecast + seasonality_forecast` | `sigmoid(classifier(concat(trend_residual, season_residual)))` |
| **Metrics** | RMSE, MAE, NRMSE, CVRMSE | Accuracy, Precision, Recall, F1, AUC-ROC |

- **Trend stack (LSTM):** Captures smooth, slowly-varying deviations вҖ” good for detecting level shifts or gradual drifts.
- **Seasonality stack (MLP-Mixer):** Captures periodic pattern deviations вҖ” good for detecting unusual daily/weekly cycles.
- **Classifier head:** A 2-layer MLP that fuses both residual streams and produces one anomaly logit per time step.

## Features

- **Hybrid Architecture:** LSTM for trend + MLP-Mixer for seasonality, with N-BEATS-style residual learning
- **Lightweight:** ~0.13M parameters вҖ” edge-deployable
- **Percentile-based labeling:** Unsupervised label generation from decomposed components
- **Point-wise detection:** Outputs one anomaly score per time step
- **Wavelet decomposition:** Cleanly separates trend from seasonal+residual structure
- **Fine-tuning support:** Pre-trained checkpoints can be fine-tuned on new buildings/domains

## рҹҸ·пёҸ Data Format

Each CSV/Parquet file should contain at minimum:

| Column   | Description |
|----------|-------------|
| `energy` | Energy consumption values (float) |
| `label`  | Binary anomaly label per point: `0` = normal, `1` = anomalous |

If `label` is not present, it defaults to all zeros (no anomalies).

### Generating Labels with Percentiles

Labels are created via `detect_anomalies.py`:

1. Decompose the series into **trend** and **seasonality**.
2. Compute upper/lower **percentile bounds** for each component.
3. Flag a point as **anomalous** if it falls outside the bounds in **either** component.

## Usage

### Training
```bash
python train.py
```
Trains the model using `BCEWithLogitsLoss`. Configuration is loaded from `configs/W_LSTMix.json`.

### Testing
```bash
python test.py
```
Evaluates on the test set, reporting per-building **Accuracy, Precision, Recall, F1, AUC-ROC**, plus a confusion matrix breakdown (TP/FP/TN/FN).

### Fine-tuning
```bash
python finetune.py
```
Fine-tunes a pre-trained checkpoint on a new dataset or domain.

## Configuration

All hyperparameters are in `configs/W_LSTMix.json`:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `backcast_length` | 168 | Input window size (7 days Г— 24h) |
| `stride` | 24 | Step between consecutive windows |
| `batch_size` | 128 | Training batch size |
| `num_blocks_per_stack` | 3 | Number of residual blocks per stack |
| `embed_dim` | 8 | LSTM embedding dimension |
| `hidden_dim` | 256 | MLP-Mixer hidden dimension |
| `ff_hidden_dim` | 256 | Classifier head hidden dimension |
| `patch_size` | 8 | MLP-Mixer patch size |
| `thetas_dim` | 8 | Basis expansion dimension |
| `threshold` | 0.5 | Classification threshold for sigmoid output |
| `loss` | `bce` | Loss function (BCEWithLogitsLoss) |
| `method_decom` | `wavelet` | Decomposition method (`wavelet` or `seasonal_decompose`) |
| `learning_rate` | 0.0001 | Adam learning rate |
| `patience` | 10 | Early stopping patience |
